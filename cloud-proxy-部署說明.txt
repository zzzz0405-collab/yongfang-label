永芳標籤 — 雲端列印代理部署（中文介面推薦：Zeabur）
========================================================

【為什麼要架代理？】
  手機 / GitHub Pages 的網頁不能直接呼叫芯烨雲 API（瀏覽器限制）。
  雲端代理只做「轉發」，列印仍是：
  手機 → 代理(HTTPS) → 芯烨雲 → 你家 WiFi 標籤機
  ※ 不需家裡電腦開著。

========================================================
方案 A：Zeabur（繁體中文介面，台灣常用，推薦）
========================================================

1. 開啟 https://zeabur.com
   - 右上角可切換「繁體中文」
   - 用 GitHub 登入

2. 點「建立專案」→ 輸入名稱（例如：永芳標籤）

3. 點「部署新服務」→「GitHub」→ 選你的 yongfang-label 倉庫

4. Zeabur 會自動偵測 Node.js。到該服務的「設定」：
   - 啟動指令（Start Command）改成：
     npm run start:cloud-proxy
   - 或維持 npm start，並在「環境變數」加一筆：
     名稱 CLOUD_PROXY_ONLY  值 1

5. 等部署完成（約 1～3 分鐘），到「網路」或服務總覽複製 HTTPS 網址
   例如：https://yongfang-label-xxxxx.zeabur.app

6. 用瀏覽器測試（應出現 JSON）：
   https://你的網址.zeabur.app/api/health

7. 在本機 index.html 搜尋 EMBEDDED_CLOUD_PROXY_URL，改成你的網址：
   const EMBEDDED_CLOUD_PROXY_URL = 'https://yongfang-label-xxxxx.zeabur.app';

8. 把 index.html push 到 GitHub，等 Pages 更新（約 1 分鐘）

9. 手機開 GitHub Pages →「印表機」：
   - 確認 API 代理網址已帶入
   - 填開發者 ID、KEY、印表機 SN → 儲存
   - 藍燈亮 → 試「API 標籤」

【Zeabur 費用】
  新帳號通常有免費額度可試用；詳情以 Zeabur 官網為準。
  用量小（只轉發列印）通常很省。

========================================================
方案 B：Render（英文介面，免費方案可試）
========================================================

  見同資料夾 render.yaml 與下方英文步驟摘要：
  1. https://render.com 用 GitHub 登入
  2. New Web Service → 選 repo
  3. Start Command：npm run start:cloud-proxy
  4. Free plan → 部署後複製 https://xxx.onrender.com
  ※ 免費版閒置會休眠，第一次列印可能多等 30～60 秒

========================================================
方案 C：阿里雲 / 騰訊雲（全中文，較進階）
========================================================

  需購買「輕量應用伺服器」或容器服務，適合已有雲端帳號者。
  在伺服器上 git clone 專案 → npm install → npm run start:cloud-proxy
  並在防火牆開放埠號、綁定網域與 HTTPS 憑證。
  一般使用者建議先用 Zeabur。

【安全說明】
  代理不儲存 ID/KEY，請求直轉芯烨雲。
  勿把 Token 寫進程式碼再上傳 GitHub。
